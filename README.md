# Інструкція до використання

1. *Запуск серверу* - для роботи мережевого режиму TCP клієнт-сервер необхідно спершу запустити клас UniversityServer (метод main).


2. *Запуск головної програми* - потім можна запускати основну програму - клас MainMenu.

***Після цього програма підвантажить наявні дані з файлу uniData.json***

3. *Авторизація* - необхідно увійти в акаунт за допомогою логіна і пароля. Якщо програма не має жодного користувача в системі, вона створить базовий акаунт sashaFlom (адміністратор).

***Примітка: якщо ввести неправильний пароль до акаунту 3 рази, користувача заблокують на час поточної сесії.***

4. *Вибір операції* - далі на вибір ідуть 4 можливих операції


   1. *операції керування даними* - (***ця опція доступна лише для ролі Адміністратор та Менеджер***) тут можна змінювати інформацію, що зберігається в системі
       1. *факультет*
      2. *кафедра*
      3. *студент*
      4. *викладач*
      5. *користувач* (***доступно тільки для ролі Адміністратор***)
    
       Для кожної з вище згаданих сутностей доступні такі дії:
      - створити
      - побачити всі
      - редагувати
      - видалити
        
       Для користувачів ще є дві додаткові опції: заблокувати та розблокувати.


   2. *пошук* - тут можна шукати

        1. студентів
   - за ПІБ
   - за курсом
   - за групою
      2. викладачів
   - за унікальним ідентифікатором
   - за ПІБ

   3. *звіти* - тут можна побачити відсортовані списки

       1. студентів
    - всі, впорядковані за курсом
    - всі, впорядковані за алфавітом
    - студенти певної кафедри:
      - впорядковані за курсом
      - впорядковані за алфавітом
      - певного курсу, впорядковані за алфавітом
    - певного факультету, впорядковані за алфавітом
        2. викладачів
    - всі, впорядковані за алфавітом
    - певної кафедри, впорядковані за алфавітом
    - певного факультету, впорядковані за алфавітом


   4. *закінчити* - ця опція завершує програму.
